我要开发一个 VS Code 扩展：支持符号链接的文件夹搜索插件

## 核心需求

1. 在 Quick Pick (Cmd+P) 中搜索工作区内的所有文件夹和文件
2. **必须支持符号链接（symlink）**，能够索引符号链接指向的目录内容
3. 智能排序：文件夹优先、精确匹配优先、前缀匹配次之
4. 缓存机制，避免重复扫描
5. 防止循环引用导致的无限递归

## 技术要点

### 符号链接处理（关键！）
```javascript
// 必须使用 fs.stat() 而不是 fs.lstat()
const stats = await fs.promises.stat(fullPath);  // 跟随符号链接
const isDirectory = stats.isDirectory();

// 使用 realpathSync 防止循环引用
const realPath = fs.realpathSync(dirPath);
if (visited.has(realPath)) return;  // 已访问过
visited.add(realPath);
```

### 配置选项
- `followSymlinks`: 是否跟随符号链接（默认 true）
- `maxDepth`: 最大扫描深度（默认 10）
- `includeFiles`: 是否包含文件（默认 true）
- `excludePatterns`: 排除的目录（默认：node_modules, .git, dist, build 等）
- `maxResults`: 最大索引数量（默认 10000）

### 命令
- `folder-search.search`: 打开搜索面板
- `folder-search.refreshCache`: 刷新索引缓存
